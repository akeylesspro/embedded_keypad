ble_dis_c.c

Purpose

Implements the Device Information Service (DIS) client. It discovers DIS characteristics on a peer and reads their values.

Main responsibilities

  • Decode DIS characteristic values from read responses.
  • Track handles discovered via DB discovery.
  • Dispatch read responses and errors to the application.
  • Queue GATT read requests using the GATT queue module.

Key behaviors

  • ble_dis_c_init registers DIS discovery.
  • ble_dis_c_on_db_disc_evt maps characteristic handles.
  • ble_dis_c_read performs queued reads.
  • ble_dis_c_on_ble_evt handles responses and disconnects.

Decoding logic

  • System ID and PnP ID decoding validate payload lengths.
  • String characteristics are passed as raw byte buffers.
  • Peripheral classification uses decode_peripheral_device_by_dis.

Dependencies

Uses DB discovery, GATT queue, and BLE services manager helpers.

Notes

Some DIS characteristics are currently disabled by commented code.
